#Introduction

TnT is a pure javascript library built on top of D3.js. It allows to build complex visualizations of trees and associated annotations. Typically, the TnT library is used by specific themes that builds final plug-ins ready to use in web applications. These themes are built to maximize reusability and many examples are provided in the themes page. The core + theme strategy converts TnT into very powerful and customizable plug-ins easy to use and re-use

Once you have built your own theme (or if you are re-using any of the themes provided by TnT), your web page should include something like this:

{% highlight html linenos %} {% endhighlight %} {% highlight javascript linenos %} var t = tnt.tree().width(620); // other methods can be included here var theme = tnt_theme(); // other methods can be included here theme (t, document.getElementById('DOM_element_id'); {% endhighlight %}

Check more examples

The first block links with the TnT core library and style sheet and one of the pre-defined themes. The first line of the second block creates an instance of the tnt object. This object is configurable via a set of methods. All these methods are explained in the library API documentation and the most important ones are explained below

A theme object is obtained in line #3. This theme object is itself a closure or code reference, that typically can be called with two parameters, the first one is the TnT object and the other one is a DOM element (normally a div element).

#The tnt object

The TnT object includes a rich API that is normally used by the TnT themes to build the final visualization. The methods defined in this object follow the method-chaining pattern. For example: {% highlight javascript linenos %} var tree = tnt.tree() .data(...) .display(...) {% endhighlight %}

By convention, this object is passed as an argument to the theme

The TnT library has methods to manipulate and visualize trees, trees + annotations or even track-based annotations without trees. Examples on how to use each component are given in the themes page

#The theme object

Themes are reusable examples of tree visualizations made with TnT. They are not mandatory, you can use the TnT library directly in your web page, but encapsulating the visualizations on themes allow them to be easily reused, so we encourage to use the TnT library through theme objects following the pattern explained in the themes page.

For example, one theme may create a tree visualization that adds tooltips to the nodes on mouse clicks, adds some options to sort the leaves based on different criteria and adds a button to export the visualization to png. All these functionalities are defined in the TnT library, but the theme only uses a set of them to create a particular plug-in

By convention, TnT themes should export one function that accepts two parameters: a TnT object and a DOM element (usually a div element where the tree visualization will be inserted. A minimal theme (just displaying the tree object) looks like:

{% highlight javascript linenos %} var tnt_theme = function () { var my_theme = function (tree, div) { // Populate the div with different elements? // Tweak and configure the tree object? // etc... tree (div); }; return my_theme; }; {% endhighlight %}

#Browser support

TnT works on HTML5, CSS3 and SVG-compliant browser, which generally means any "modern browser". The TnT library and themes are tested regularly against Chrome (Chromium) and major changes are regularly tested in recent versions of Firefox, Safari (Webkit) and Opera. It has been also tested in IE versions 9+. If you find any problem, please let me know.

The TnT library depends only on the D3.js library. Check its browser's compatibility note for more information. TnT themes may depend on other libraries and components for enhanced functionality. Check the documentation of the corresponding TnT theme for more information
